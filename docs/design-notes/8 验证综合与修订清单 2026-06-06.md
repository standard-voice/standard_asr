# Standard ASR 规范验证综合 + 修订清单 2026-06-06

> 4 路独立 subagent 验证**写出来的 normative 规范**(不再是方向):V1 音频/打包/配置(§AI/§DEP/§IC)、V2 能力/runtime/结果(§C/§R/§TR)、V3 streaming(§ST)、V4 跨段+三基准端到端。
>
> **结论:架构验证通过、零方向反转、三基准端到端可驱动**(OpenAI 批✓/流被 1 项挡、ElevenLabs ✓✓、Qwen 批✓/流需 1 项 schema 补)。约 29 处**外科式修订(无重设计)**。多路撞到的 = 高置信。

## 晋级判定
- **§DEP**:直接晋级(numpy 事实核对无误)。
- **§AI / §C / §R / §TR / §IC**:修下列后晋级。
- **§ST**:修 + **再过一轮 gate 复审**(早先承诺过,V3 也明确要求)。

## A. 跨段收敛(多验证撞到,最优先)
1. **旧草稿未标取代 + 算法滞留**(V2-B1, V1-B2, V4-C5):`设计规范.md` 开头 ~300 行旧草稿未标「已被取代」;`max_candidate_languages` 双家(旧 Properties + 新能力节点);`effective_language`/`effective_candidate_languages` 算法仍引旧字段名,而它们是**真正会跑的逻辑**。→ 给旧草稿打 banner / 挪到笔记;把两个算法用新路径(`capabilities.<mode>.language.*`、`…candidate_languages.constraints.max`)**重述为 normative**;改所有 `max_candidate_languages` 引用指向能力节点。
2. **流式轴双拼写**(V1-M2, V4-C2):§C.2 `streaming_input/streaming_output` vs §ST.2 `supports_streaming_input/supports_streaming_output`(被废的 `supports_` 前缀)。→ §ST 统一为裸名。
3. **§ST↔§C 能力对不齐**(V2-B3, V3-C6, V4-C3):§C.2 缺 `re_segments`/`right_context`/`mutable_mid_stream`/`word_stability`,§ST 却在用。→ 补进 §C.2(`right_context` 按 I/O 边界值可放 Properties);明文 `no_interim ≡ emits_partials.supported=false`。
4. **时间原点措辞不一**(V3-C4, V4-C6):§TR「提交音频起点」vs §ST「会话起点」。→ 统一定义「音频第一采样 = t=0(非 wall-clock 会话开启)」;`audio_processed_until`/`Segment.start/end` 同一原点。

## B. §AI / §IC (V1)
5. **B1 effective_language totality**:IC.6 谓词收紧——「有语言轴(`selectable_languages` 已定义,含单语言/`["auto"]`)⇒ `default_language` 必给(可 `auto`)」;无语言轴 ⇒ algorithm 在 step2 前短路 `None`。
6. **M1**:旧文 line 119「对所有引擎强制 default_language」vs IC.6 条件式,自相矛盾 → 统一为 #5 的规则。
7. **M3 SSRF**:私网/环回/link-local 拒绝 + HTTPS-only + 重定向上限 提到 **v1 透传路径**(MUST),不只 v2 自取。
8. **M4 self_resamples 静默漏洞**:`self_resamples=true` 但喂入率 ∉ `accepted_sample_rates` 时 → `accepted_sample_rates` **始终权威**,标准 MUST 重采样或定义错误,不静默透传。
9. **m2**:`supported_input_formats` 是孤儿字段(AI.10 列出但无处用)→ 接进 encoder/passthrough 决策(encoder 选 engine 接受的容器)或删。

## C. §C / §R / §TR (V2)
10. **B2**:`engine.supports()` 路径根定死(从 mode 域 / 顶层正交能力起,**不带** `capabilities.` 前缀);改 §C.6 示例。
11. **B4**:`mutable_mid_stream` 许诺的 `update_guidance()` §ST 未定义 → **【判断:v1 砍,保留 capability flag、不承诺方法】**(与 cut `target_latency` 运行时调整一致;AssemblyAI/Deepgram-Flux 的中途改词留 v2)。
12. **M1**:diarization 三链补一句——`Segment.speaker` 填充 iff `<mode>.diarization.supported`;v1 **请求**走 `provider_params`(结果 shape 已预留)。
13. **M2**:`word_timestamps` 能力 bare `{supported}` → `{supported, granularities:[word,segment,char]}`(否则测不出只支持 segment 的引擎,枚举意义落空)。
14. **M3**:`provider_params` **wire 错误模型**——`InvalidProviderParamError → HTTP 422 / WS error frame {code,field,reason}`(跨语言)。
15. **M4**:streaming best_effort/降级 diagnostic **无投递通道**(§ST 事件无 diagnostic 载体,§TR.diagnostics 是批量)→ 挂到 `progress`/`error` 事件或随 #11 一并消除。
16. **M5**:多通道「时序合并顶层 text」与 §TR.2「跨段单调」在双声道同时说话时**互斥** → 改「**每通道**单调;顶层 stable-sort by `start`,tie by `channel`,跨通道允许 `[start,end]` 重叠」。
17. enum/mode 节点的派生 `supported` MUST 出现在 canonical JSON(跨语言探针)。

## D. §ST streaming (V3) — 最重 + gate 项
18. **C2(gate)**:冻结前缀保留写在「音频位置」空间,但 `stable_until` 是 codepoints,**跨 merge 无桥** → 用 codepoint 空间重述(old→new 偏移映射规则:跨 merge 的冻结前缀 codepoint-for-codepoint 仍是 new 的冻结前缀)+ **merge/split 工作示例**。
19. **C1 核心 reduce 自洽**:`new_ids` 可先以 `partial` 到达(非必 final);reduce 改「**原子替换** [old_ids]→[new_ids],渲染 new 的当前文本」,非「先删后等」;`old_ids ∩ new_ids` 禁用(id 一旦 supersede 即退休)或显式定义。
20. **C3 背压**:partial 合并 MUST 被 `supersede`/`closed`/`final` **作废**;不跨终态/supersede 边界投递某段 partial(否则复活陈旧段)。
21. **B-NEW-1 重连按源分类**:rewindable(file/array/server-buffered)→ rolling buffer 重喂、gap 内部;**live/不可回放(mic)→ 不承诺无缝**,超缓冲的样本真丢,`progress{reconnect,gap,recoverable:false}` 标「内容丢失」;`feed()` 加 source-class 概念以界定缓冲上限。
22. **B-NEW-2 段间时序**:新 `segment_id` 可在别段 `final`/`closed` 后开 `partial`(interim→commit→新段,所有云 WS 通用;否则挡 EL)——明文 MUST 允许。
23. **B-NEW-4 / C7**:无 ts/right_context 引擎(Qwen)明文 escape——MUST 报 `stable_until=0`,语音 agent 前缀不可用;**`emits_partials` 与 `streaming_output` 解耦**——`streaming_output=true` 只表「输入结束前出结果」,`emits_partials=false` 引擎只出段末 `final`(no_interim 档:FireRedASR2S/SenseVoice+VAD),修 ST.2「output=true MUST 出 partial」的过强句。
24. **C5 stable_until 字素边界**:`stable_until` MUST 落 grapheme-cluster(UAX#29)边界(CJK/印度系/emoji——标准明示的核心语言用户;codepoint 切碎簇 = 可见前缀回退)。
25. **B-NEW-3**:`segment_id` 铸造规则(引擎给则用、无则适配器按定义合成)——避免 id-keying 切引擎崩。
26. **m-NEW-1 状态转移表**:`open→{partial,final,supersede}`、`final→{supersede,closed}`、`closed→{}`;`partial`/`final` after `closed` 或同 id `final` 后 = MUST NOT;`progress` 任意非终态合法(含 recoverable-error 后)。
27. **m-NEW-2/3/4**:`feed()` 模式下任何手动 `send_audio`/`end_audio` raise(ST.1 胜 ST.7 幂等);sync 桥事件队列同 ST.7 背压;`closed` 可**原位改写**同 id 文本(`re_segments=false` 只管边界、不管原位重格式化)——核心 app 须把 `closed` 当替换非追加。

## E. §AI / 跨段 (V4)
28. **C-1(blocker)**:`start_transcription(audio_format, params)` **无音频参** → 加 `audio: AudioInput | None`(与 `audio_format`+`send_audio` 互斥),解 OpenAI-SSE「整段输入+流式输出」(§AI.1 已假定它存在)。
