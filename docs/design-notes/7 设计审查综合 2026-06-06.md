## 0. 一句话结论

**所有 10 个设计方向全部通过对抗审查，零反转。** 普遍症结只有一个：**每个决策都是「方向 (direction)」而非「规范 (specification)」**——拍了容器，却把决定可移植性/正确性的硬性规范留给了实现者。修订工作几乎全是「把方向硬化成规范」+「堵住几条静默失败路径」+「砍掉 v1 过度设计」，**不是重新设计**。

## 1. 跨决策元模式（最高杠杆）

### 1.1 零方向反转
判别联合音频输入、能力协商、稳定前缀 streaming、两方法 API、结果 shape 恒定、可移植集+逃生舱、numpy 宽兼容、层级化能力、Scheme A 命名——**没有一个被推翻**。被否的备选（仅数组、随 flag 变形、单 prompt、强制 numpy2、prototype 半双工 streaming）都被独立验证为正确地否决。

### 1.2 反复出现的陷阱类别
| 类别 | 实例 |
|---|---|
| **静默失败 / 假可移植**（违反项目自身「日志不是 API 合约、静默错误是大忌」原则） | 裸数组假定 16k(D2)、单 `prompt` 假可移植(D6)、多通道顶层 text 歧义(D8)、null 歧义"不支持 vs 支持但空"(D8)、凭证序列化泄漏(D7)、重采样无抗混叠(D3) |
| **命名未定义的机制** | 转换矩阵(D1)、能力 schema/生命周期/演进(D5)、段生命周期状态机(D10)、env 变量命名(D7)、provider_params 错误模型(D6)、判别器(D7) |
| **安全漏洞**（10 年标准不能留） | AudioUrl SSRF(D1)、凭证泄漏(D7) |
| **事实错误** | OpenAI SSE 整段输入也流式吐 partial(D10.8)、numpy 1.24 无 3.13 wheel + clip/astype 行为变化(D4) |
| **跨决策不一致** | D5 能力 vs D10.6 StreamingCapabilities、D2 采样率字段 vs 既有 BaseProperties、D7 default_language vs 已定稿语言设计、D8 时间单位只在差距分析、D6↔D7 init/runtime 边界 |
| **v1 过度设计** | D2 六字段→二、D10 merge/split→一个 supersede + 砍运行时延迟/edit-ops、D5 满地 metadata、D6 diarization/entities 先 provider_params、D8 speakers[]/entities 顶层、D7 凭证 profile/轮换 |

### 1.3 一个反复缺失的核心概念：**applicability predicate（适用性谓词）**
D2、D5、D7 各自独立撞到同一个缺口：**「可选标准字段/能力应当『适用才存在』，由 capability 门控；存在⇒适用，缺失⇒不适用」**。统一引入这一概念可一次性解决：
- D7 的 `device`/`default_language`/`download_root` 对云引擎的歧义（用 capability-bearing config **mixin**：字段在模型里出现 ⇒ 适用，auto-UI 据此渲染正确表单）；
- D5 的「缺失能力键 = 不支持」(fail-closed)；
- D2 的采样率字段该放哪、何时适用。
**修订时把它写成一条贯穿 D5/D7/D2 的规则。**

## 2. 必须修的事实错误（最优先）

- **D10.8 — OpenAI SSE 描述反了**：`stream=true` 上传整段文件**会在转写完成前就流式吐 `partial`**（已核 OpenAI cookbook）。我写的「`end_audio` 前无 partial」是错的。正解：**输入时序 ⊥ 输出时序**——OpenAI Audio = `streaming_input=false` 但 `streaming_output=true` **且有 partial**。重写 D10.8 的规范句。
- **D4 — numpy floor 错**：numpy 1.24/1.26 **无 Python 3.13 wheel**（3.13 支持始于 numpy 2.0）。「核心 ∩ FunASR<2 = 1.24–1.26 可解」在 3.13+ 上**为空**。改为 interpreter-conditional bound（`numpy>=1.26; python<"3.13"` / `numpy>=2.1; python>="3.13"`），并改正那个示例。
- **D4 — `clip`/`astype` 非"行为一致"**：numpy 2.0 改了 `clip`（None 处理 + OOB 整数边界 OverflowError）和 `astype`（OOB float→int、`copy` 关键字）。不能宣称"1.24→2.x 一致"；加 **ruff NPY201** + warnings-as-errors 双版本 CI；编码路径强制「先 clip 再 cast」、禁 `copy=False`。

## 3. 安全（不可留到 v1 之后）

- **D1 AudioUrl SSRF**：引入 URL 透传却无 fetch 策略。两类 fetcher：引擎/云自取（标准只需 HTTPS + scheme/host 校验后转发）vs **标准自取**（URL→数组引擎时我们 fetch，直接 SSRF 洞——`169.254.169.254`、`localhost`、`file://`、重定向进内网）。**v1 决定：`AudioUrl` 仅作"透传给接受 URL 的引擎"，标准自取 URL→本地解码这一格延后**，且即便透传也要 HTTPS-only + 私网/环回 IP 段默认拒绝 + 重定向上限。
- **D7 凭证序列化泄漏**：`api_key` 作为普通 `str` 在 `frozen=True` + UI-discoverable + 序列化（`/v1/models`、持久化、telemetry）的模型上，会进 `model_dump`/OpenAPI schema/`repr`/日志。「禁入日志」是文档而非类型。**强制 `SecretStr` + 脱敏序列化 + REST write-only + secret-field 标记给 auto-UI**。把 `base_url`/`region`（非密）与 `api_key`（密）分两类。

## 4. 跨决策一致性（修订时必须缝合）

1. **能力系统单一真相源**：D10.6 的 `StreamingCapabilities` 必须**就是** `capabilities.streaming.*`（D5），不能两套 schema。D5 落第二就 subordinate 到对方。
2. **采样率字段归位**：D2 新造的 `accepted_sample_rates/channels/encoding` 与既有 `BaseProperties.supported_sample_rates/supported_channels/audio_dtype` 重复——按项目自身边界规则（静态身份/边界值→Properties），把采样率元数据放 Properties，只有行为性 `self_resamples` 放 capability。**别造平行词汇**（正是 D5 刚消灭的反模式）。
3. **default_language 矛盾**：D7 说标准字段「对所有引擎非强制」直接冲突已定稿语言设计的「强制默认值」（`effective_language` 算法依赖它恒有值）。用 §1.3 的适用性谓词解决：**适用（声明语言能力）⇒ 必填或标准默认（可为 `auto`）**。
4. **时间单位进 D8**：`start/end = float 秒、原点=音频起点`（与 D10.7 流原点同），只在差距分析出现过，必须写进 D8 本身。
5. **init/runtime 边界**（D6↔D7）：写一条对称规则 + tie-breaker：**「能按请求变 ⇒ runtime（provider_params），不进 init」**；模型选择走 entrypoint preset，不是 init `model` 字段。
6. **结果子模型共享**：`TranscriptionEvent.segment/.words` 与 `TranscriptionResult` 用**同一** `Segment`/`Word`（流式专属 `stable_until` 加在事件包装层）；加 `session.result() -> TranscriptionResult` 可归约性。

## 5. 逐决策 punch-list（仅列 blocker + 关键 cut；详见对话）

**D1 音频输入** — blocker：判别器未定义且歧义（`str`=路径/URL/base64、`bytes`=编码/裸PCM → 用 `AudioPath/AudioUrl/AudioBytes/AudioArray/AudioBase64` 标签类型，裸类型仅作已定义的便利强制转换）；转换矩阵须穷举（含 **本地文件→URL-only 引擎死角**，比数组→URL 更常见）；死角需 typed 谓词 `negotiate()/can_accept()` + `IncompatibleAudioInputError` + 引擎 card 字段；数组→文件 encoder 须落地（内存 BytesIO、WAV-16、mono、**预检 `max_file_size`** 防 OpenAI 25MB 悬崖）；SSRF（§3）。**Cut**：标准自取 URL→本地解码、upload-broker、多通道/高位深 encode、base64-native（折成 coercion）。

**D2+D3 采样率·重采样** — blocker：裸数组假定 16k 静默腐坏（strict 要求 `(array,sr)`，best_effort 假定 16k **必带 diagnostic**）；8k 电话路由缺失（8k=独立模型→**preset 选择**，接 GAP-20，标准**禁止**升采样原生 8k；server-routed 用 `domain` 提示）。major：24k realtime 与"避免重采样"矛盾（点名它为"标准重采样"的典范情形，定 16k→24k 谁来做）；`resampling` 四值无权威语义（→ `self_resamples: bool` + "false⇒标准必须交付可接受率，否则定义错误"）；**重采样器必须抗混叠**（FFT-based，numpy 自带，禁裸线性/抽取；结构化 `resampled_with=fallback`）。**Cut**：六字段→`accepted_sample_rates`+`required_input_sample_rate`+`self_resamples`+元数据；许可证用 clean-room FFT，**排除 SoX/soxr(LGPL/GPL)、2016 前 libsamplerate 的 vendoring**（soxr 仅作 `[audio]` 依赖）。

**D4 numpy** — blocker（事实，§2）：floor 错、clip/astype 行为变化。major：进程隔离逃生舱只是口号（无 `engine_id→endpoint` 寻址、选了最重的 FastAPI、忽略音频跨进程序列化成本、且强迫 app 改代码违反 P.1.1）→ 指定寻址、推荐 **subprocess+UDS+shared_memory** 作更轻默认、承诺薄 **isolation shim**（fast-follow）。新增 **`standard-asr doctor`** 只读诊断检测插件 numpy 冲突。**Cut**：可选 numpy（留 hard dep）、任何上限 cap、FastAPI 作首选隔离。

**D5 能力系统** — blocker：静态 vs 实例生命周期未定（回归了 idea_docs 已有的 `ClassVar` 保证）→ **`DeclaredCapabilities`（ClassVar、免实例化/免鉴权可发现）+ 可选 `effective_capabilities`（只能收窄，`effective ⊆ declared`）**；未知键/向前兼容未定（10 年症结）→ **fail-closed（缺失=不支持）+ 宽容解析未知键 + major 内只增不改 + `x_*` 实验命名空间**替代被删的 extra→标准化路径。major：节点 shape 含混（flag/bounded/enum 三态，每节点须可取 `.supported`）；域未定义（**封闭 mode 域 batch/streaming，保留 job**；mode 内 feature 标准所有）；能力↔属性边界被 `约束/metadata` 重新搅浑（`max_*` 选一处家）；app 访问 `engine.supports("streaming.word_timestamps")` fail-closed + 统一 JSON。**Cut**：满地 `metadata`、开放域 `含`。

**D6 runtime 参数** — blocker：标准/provider 线无规则（加 4 条准入测试 + 封闭 v1 可移植表）；**单 `prompt` 假可移植**（Whisper initial_prompt≠Qwen context≠EL keyterms≠hotword-ID → 拆 `prompt`(free-text)+`phrase_hints`(boost)，或 **v1 全进 provider_params**）；strict/best_effort×provider_params×pydantic 错误模型未定（`extra="forbid"`、校验抛错独立于 best_effort、type-to-engine 保 swap 安全）。major：auto-UI↔非可移植矛盾（隔离+标注"锁定 {engine}"、默认折叠）；跨语言须 **JSON Schema over the wire**（非 Python-only）；治理无定义（≥N 引擎等价⇒minor 版本，单向）；"禁顶层字段"无强制（**封死请求类型**+合规测试）。**v1 可移植集 = `language`+`candidate_languages`+`word_timestamps`(粒度枚举) only**，其余 provider_params。

**D7 Init Config** — blocker：`engine` 判别器可碰撞且与身份混淆（→ **entrypoint 派生 engine_id**、registry 查找而非跨插件 Union、检测重复 id）；凭证无 secret 机制→序列化泄漏（§3）。major：env 命名歧义（normalization + 每标准字段一约定 + 优先级 + 多账户 profile 段）；**default_language 矛盾**（§4.3）；适用性信号缺失（§1.3 mixin）；init/runtime 边界（§4.5）；下载/缓存字段无家 + lazy-load 纯度不变量。**Cut**：多账户 profile 系统（留 hook）、bundle 形状标准化（只给 nested submodel + 路径 helper）、凭证轮换/STS、宇宙级 Union 类型。

**D8 结果 shape + D9 命名** — blocker：多通道顶层 text 契约未定（顶层=**全通道时序合并**，永不只 channel-0；`channels: list|None`；Segment/Word 带可选 `channel`）。major：null 歧义（capability=支持；`None`=未请求、`[]`=请求但空）；SRT/VTT 替代路径（核心 `to_srt(result)/to_vtt(result)`，provider 格式仅作 `extra` 透传）；diarization shape **现在就预留** `speaker` 于 Segment(权威)+Word；时间单位进 D8（§4.4）；流批子模型共享 + `session.result()`（§4.6）。**Cut**：diarization/entities 特性（保留 shape）、顶层 `speakers[]`、provider 格式字段、char 级时间戳/logprob 向量。**D9**：稳；两句澄清——始终用限定词 `transcription-session`（避与 auth/Realtime "session" 撞）、`send_audio/end_audio` 是 RPC 子动作非子资源。

**D10 streaming** — blocker：D10.8 事实错（§2）；`supersede/merge/split` 只命名未规范且**与"按 segment_id 提交"核心契约冲突**（核心-only app 忽略它会留陈旧段）+ 是冻结前缀**回退向量**（违反头号保证，除非加保留规则）。major：`closed` 不可被 supersede（须硬不变量）；`target_latency` 中途调整**无控制通道**（砍到构造期）；DSM 时间游标/心跳**无事件类型**；`feed()` vs 手动 single-owner 规则；sync 桥适配器契约；透明重连过度承诺（藏了滚动音频缓冲 + 改"会话存活"而非"app 无感"，发 `reconnect(lossy,gap)`）。`stable_until` 单位（→ codepoints）。**Cut**：运行时延迟调整、edit-ops、merge/split 三原语→一个 `supersede(old_ids[]→new_ids[])`、繁复 `closed` 机制、有状态 DSM 无缝重连。

## 6. 提议的 v1 范围（需要你拍板）

审查一致主张：**ship 一个『小而真』的核心，靠治理增长，而非 ship 一个『大而撒谎』的核心**。据此提议——

**v1 ship（真核心）**：
- 音频：判别联合（标签类型）+ 协商 + 透传 + 解码(`[audio]`) + 数组→mono-WAV encode(带 size 预检)；`AudioUrl` 仅透传。
- 采样率：canonical 16k mono；率随音频；裸数组 strict 报错/best_effort 带 diagnostic；`self_resamples` + 抗混叠 fallback；8k=preset。
- 结果：恒定 schema + 时间单位(秒) + 多通道时序合并 + null 规则 + 核心 `to_srt/to_vtt`；预留 `speaker`/`channel` shape。
- 能力：`DeclaredCapabilities`(ClassVar) + fail-closed + 封闭 mode 域 + 三态节点 + `engine.supports()`。
- runtime：可移植集 = `language`+`candidate_languages`+`word_timestamps`+**`prompt`(free-text 软提示)+`phrase_hints`(词条 boost，无权重)**；其余 → typed `provider_params`(JSON-Schema-over-wire、extra=forbid、标注非可移植)。**[用户定调 2026-06-06：prompt 太重要，留核心，按 free-text/phrase/resource-id 三分而非塞 provider_params；具体方案审查中]**
- Init：entrypoint 派生 `engine`、`SecretStr` 凭证、env 回退、适用性 mixin、lazy 纯度。
- numpy：interpreter-conditional bound、无上限、ruff NPY201、`doctor` 诊断。
- streaming：`partial/final` + `segment_id` + 保守 `stable_until`(codepoints, 绑 right_context) + `end_audio` + `error(code,recoverable,retriable_after)` + `done` + 两级终态**标志** + 一个 `supersede` + 正交 in/out 能力；session 拥有 pump + 标准 sync 桥；`reconnect(lossy,gap)` 信号。

**v1 defer（保留 shape/hook，加时为 additive-minor）**：
- diarization、entities/redaction、多通道**分离**（保留字段）、char 时间戳/logprob、`phrase_hints` 的**权重**（先进 provider_params）、预注册 vocab/model ID（→ init config）、运行时延迟调整、edit-ops、无缝 DSM 重连、多账户凭证 profile、bundle 形状标准化、可选 numpy、进程隔离 shim（先文档 + `doctor`）。

## 8. `guidance`（引导）家族 —— 待 Batch 2(D6) 落稿（2 路 subagent 已定向）

> 用户定调「prompt 是核心、不进 provider_params」后，2 个独立 subagent（命名 + 抽象压测）的结论。Batch 2 据此落 D6 + 写 `设计规范.md`。

- **命名 = `guidance`**（取代 `conditioning`）。理由：`conditioning` 在 ML 恰指「训练期、写死、确定性」——与「可选/best-effort/非绑定」**语义相反**，且噪音多（健身/空调/操作性条件反射）；`guidance` 是推理期、可调、非强制的那一面（classifier-free guidance）。次选 `biasing` 否决（与 `bias_resource` 撞名 + `bias` 公平性负面色彩）；`adaptation` 太像 Google 品牌词。
- **结构 = 扁平字段 + 共享契约 + capability 子树**（**非** sub-object，**非** typed-item list——后者有 oneof null 歧义 + 丢可发现性）。`params.prompt` / `params.phrase_hints` 扁平；`capabilities.<mode>.guidance.<channel>`（mode-scoped，复用 D5 fail-closed）。
- **v1 channels**：`prompt`（free-text，**合并** OpenAI prompt + Qwen/火山 `context`）+ `phrase_hints`（**无权重** boost 集；bounds 进 capability node，仿 candidate_languages.max）。
- **3 处收紧**：① `bias_resource` → **init config**（账户级注册资源、绑定性、非 per-request hint）；② `context` v1 **并入** `prompt`（同一可观测契约，未来出现机制独立的语义上下文再 additive 拆）；③ namespace 降级为「契约+命名约定+capability 子树」。
- **共享契约**（normative）：每个 channel = optional · best-effort · 非绑定 · **正极性** · capability 协商 · **永不静默降级**（strict 报错 / 结构化 diagnostic）。
- **不对称未知处理**（JSON Schema $vocabulary / LSP 教训）：引擎收到**未知 channel = fail-closed**（不静默接受）；app 收到**未知 capability 键 = 容忍忽略**（向前兼容）。
- **降级目标**：唯一通用文本 channel；`on_unsupported="degrade_to_prompt"` **opt-in、单向（rich→prompt）、必发 diagnostic**，默认 fail-closed（直接序列化短语进 prompt 会掉精度，故不可自动）。
- **扩展机制**：`x_<vendor>_<channel>` 实验命名空间 + 文档化提升路径（提升后**去前缀**，RFC6648）；新 channel = additive-minor；**无 per-channel version**（D5 已杀）；channel 名一旦定义不复用（protobuf 教训）。
- **明确路由出去（不进 guidance）**：verbatim/disfluency/标点/ITN/profanity-**filter** → 标准 runtime **三态 flag**；profanity-**mask**/entity-redaction/格式化(JSON/会议纪要) → **result/format model**；自由文本**系统指令** → 出 ASR 范围（防 namespace 变指令垃圾桶）；domain 提示（医疗/电话）→ **entrypoint preset**。
- **reserve（additive-later）**：`context` 独立 channel、`phrase_hints` 权重（先 provider_params）、`pronunciation_hints`、`negative_phrases`/suppression（**须先给契约加 `polarity` 轴**）、音频载荷类（音频示例 / few-shot 对 / 说话人注册 → **复用 audio-input 联合的独立家族，不进 guidance**）。
