## 2. Provider 详述

### OpenAI Audio API（重点）

**来源。** 官方指南确认 STT endpoints 为 `transcriptions` / `translations`，模型包含 `whisper-1`、`gpt-4o-mini-transcribe`、`gpt-4o-transcribe`、`gpt-4o-transcribe-diarize`，文件上传 25 MB，支持 `mp3/mp4/mpeg/mpga/m4a/wav/webm`，并指出 Realtime transcription 适合麦克风/通话实时流（https://developers.openai.com/api/docs/guides/speech-to-text，2026-05-09，L668-L684）。官方 Python SDK clone：`reference/openai-python` commit `38d75d74a5626472cd7d1be9705ea8aba29a6b22`。

**3.1-3.2 基础 / 初始化。** 云 REST API，认证为 HTTP Bearer API key（https://developers.openai.com/api/reference/overview，2026-05-09，L845-L854）。SDK client 初始化通常只需 API key/base_url/timeout 等通用参数；转写模型、语言、prompt、format 在调用时提供。连接复用由 HTTP client/SDK 管理，未在音频接口文档中找到线程安全承诺。

**4.1 multipart 字段。** `POST /v1/audio/transcriptions` SDK 字段来源：`reference/openai-python/src/openai/types/audio/transcription_create_params.py:L22-L172`。必填 `file`，格式 `flac/mp3/mp4/mpeg/mpga/m4a/ogg/wav/webm`；必填 `model`，允许 `gpt-4o-transcribe`、`gpt-4o-mini-transcribe`、`gpt-4o-mini-transcribe-2025-12-15`、`whisper-1`、`gpt-4o-transcribe-diarize`；可选 `chunking_strategy`（`"auto"` 或 `server_vad` 对象，含 `prefix_padding_ms/silence_duration_ms/threshold`）、`include[]`（当前确认 `logprobs`）、`known_speaker_names[]`、`known_speaker_references[]`、`language`（ISO-639-1）、`prompt`、`response_format`、`temperature`、`timestamp_granularities[]`、`stream`。`POST /v1/audio/translations` 字段来源：`reference/openai-python/src/openai/types/audio/translation_create_params.py`：`file`、`model=whisper-1`、`prompt`（应为英文）、`response_format=json/text/srt/verbose_json/vtt`、`temperature`。

**4.2 response_format。** SDK 定义 `AudioResponseFormat = Literal["json","text","srt","verbose_json","vtt","diarized_json"]`（`reference/openai-python/src/openai/types/audio_response_format.py:L1-L7`）。本报告模型总集复刻为 `OpenAITranscription`、`OpenAITranscriptionVerbose`、`OpenAIDiarizedTranscription`、`OpenAITranslation`、`OpenAITranslationVerbose`；`text/srt/vtt` 为 `text/plain` 字符串载荷，无法用 JSON object 表示。重要冲突：官方指南说 `gpt-4o-transcribe` / `gpt-4o-mini-transcribe` 支持 `json` 或 plain `text`（https://platform.openai.com/docs/guides/speech-to-text，2026-05-09），但 SDK 注释称二者 only supported format is `json`（`transcription_create_params.py:L90-L96`）；需要进一步确认实际服务端行为。

**4.3 stream=true SSE。** SDK 的流式事件 Union 为 `TranscriptionTextSegmentEvent | TranscriptionTextDeltaEvent | TranscriptionTextDoneEvent`（`reference/openai-python/src/openai/types/audio/transcription_stream_event.py:L13-L15`）。`transcript.text.delta` 含 `delta/type/logprobs?/segment_id?`（`transcription_text_delta_event.py:L22-L45`）；`transcript.text.done` 含 `text/type/logprobs?/usage?`（`transcription_text_done_event.py:L51-L72`）；`transcript.text.segment` 用于 diarized streaming，含 `id/start/end/speaker/text/type`（`transcription_text_segment_event.py:L10-L31`）。未找到独立的 `transcript.logprobs.delta` 或类似事件；logprobs 作为字段内联。

**4.4 模型差异。** `whisper-1` 支持传统 `json/text/srt/verbose_json/vtt`，不支持 streaming，`stream=true` 会被忽略（`transcription_create_params.py:L144-L169`）。`gpt-4o-transcribe` / `gpt-4o-mini-transcribe` 支持新 token usage/logprobs；`include[]=logprobs` 仅在 `response_format=json` 且上述新模型下可用（`transcription_create_params.py:L47-L57`）。`timestamp_granularities[]` 在 SDK 注释中要求 `verbose_json`，但又说新模型 format 仅 `json`，存在待确认矛盾（L90-L115）。`gpt-4o-transcribe-diarize` 支持 speaker annotations，需 `diarized_json` 才能拿 speaker segment，且长于 30 秒需要 `chunking_strategy`；该模型不支持 `prompt`、`logprobs` 或 `timestamp_granularities[]`（`transcription_create_params.py:L37-L57,L81-L115`；https://platform.openai.com/docs/guides/speech-to-text，2026-05-09）。

**4.5 Realtime 关系。** Realtime transcription 事件名不同：`conversation.item.input_audio_transcription.delta`、`.completed`、`.failed`；delta/completed payload 见 `reference/openai-python/src/openai/types/realtime/conversation_item_input_audio_transcription_delta_event.py:L12-L40`、`completed_event.py:L60-L98`、`failed_event.py:L11-L47`。Realtime 文档/SDK说明 transcript 是独立 ASR，可能与模型对音频的语义理解不同；Standard ASR 应把 Audio API SSE 与 Realtime event stream 作为两套 adapter。

**3.3-3.10 设计记录。** 输入是 multipart file，不接受 URL；语言是 ISO-639-1，省略可自动；非流式 schema 中 `segments` 时间戳为秒，`words` 为秒；支持 prompt 作为风格/上下文续写，prompt 需匹配音频语言（`transcription_create_params.py:L81-L87`）。不可调和点：同一 endpoint 按 model 改变可用 response_format / streaming / diarization / prompt 行为，标准需要 capability negotiation。

### Groq

来源：官方文档 https://console.groq.com/docs/speech-to-text（2026-05-09）与 SDK clone `reference/groq-python` commit `fcdebf0b4318b651e42dc4ff67c859c525cbf0bb`。`TranscriptionCreateParams` 定义模型 `whisper-large-v3` / `whisper-large-v3-turbo`，输入 `file` 或 `url` 二选一，格式同 OpenAI 常见音频格式，`response_format` 仅 `json/text/verbose_json`，`timestamp_granularities[]` 仅在 `verbose_json` 下使用（`reference/groq-python/src/groq/types/audio/transcription_create_params.py:L13-L175`）。语言列表是 Whisper ISO-639-1 风格并含 `yue`（同文件 L27-L137）。翻译接口也支持 `file` 或 `url`，response_format 同样为三种（`reference/groq-python/src/groq/types/audio/translation_create_params.py:L13-L53`）。

未找到：Groq ASR 的 SSE/Realtime 增量协议、speaker diarization、word confidence 之外的完整错误模型。本次确认 Groq 是 OpenAI-compatible 的子集，不应默认继承 OpenAI 的 SRT/VTT/stream/diarized_json。

### ElevenLabs

来源：官方文档 https://elevenlabs.io/docs/api-reference/speech-to-text/convert（2026-05-09）与 SDK clone `reference/elevenlabs-python` commit `2e0470933dfe066b15ec69d78750471abbc9678b`。`convert()` 支持同步返回或 `webhook=true` 异步（`reference/elevenlabs-python/src/elevenlabs/speech_to_text/client.py:L258-L260,L303-L307`）。认证为 API key，SDK client 可复用。

输入字段：`model_id`、`file`、`language_code`、`tag_audio_events`、`num_speakers`、`timestamps_granularity`、`diarize`、`diarization_threshold`、`additional_formats`、`file_format`、`cloud_storage_url`、`source_url`、`webhook`、`temperature`、`seed`、`use_multi_channel`、`entity_detection`、`no_verbatim`、`detect_speaker_roles`、`entity_redaction`、`entity_redaction_mode`、`keyterms` 等（`client.py:L245-L257,L264-L337`）。`file_format=pcm_s16le_16` 要求 16-bit PCM、16kHz、mono、little-endian；`cloud_storage_url` 要求 HTTPS 且文件小于 2GB；`use_multi_channel` 最多 5 channels 并独立计费；`num_speakers` 最多 32；`keyterms` 最多 1000 个，每个小于 50 字符且最多 5 个词（`client.py:L279-L337`）。

返回：单通道 `SpeechToTextChunkResponseModel` 含 `language_code/language_probability/text/words/channel_index/additional_formats/transcription_id/entities/audio_duration_secs`（`speech_to_text_chunk_response_model.py:L13-L66`）；词对象含 `text/start/end/type/speaker_id/logprob/characters`（`speech_to_text_word_response_model.py:L12-L53`）；多通道返回 `transcripts[]/transcription_id/audio_duration_secs`（`multichannel_speech_to_text_response_model.py:L11-L32`）。

第二轮补查确认 realtime Scribe：`ScribeRealtime.connect()` 支持 URL-based 与 manual audio chunks，两种都要求 `model_id`；manual 还要求 `audio_format` 与 `sample_rate`（`reference/elevenlabs-python/src/elevenlabs/realtime/scribe.py:L102-L128,L181-L229`）。realtime 音频格式枚举为 `pcm_8000/16000/22050/24000/44100/48000` 与 `ulaw_8000`，commit strategy 为 `vad` 或 `manual`（`scribe.py:L22-L41`）。连接地址为 `/v1/speech-to-text/realtime`，鉴权 header `xi-api-key`；URL 模式用 ffmpeg 转 `s16le`、16k、mono（`scribe.py:L244-L320,L370-L407`）。客户端发送 JSON `message_type=input_audio_chunk`，字段 `audio_base_64/commit/sample_rate/previous_text`；manual commit 发送空音频且 `commit=true`；连接关闭后不可复用（`reference/elevenlabs-python/src/elevenlabs/realtime/connection.py:L137-L235`）。

Realtime 服务端事件 Union 已在 SDK 定义：`session_started`、`partial_transcript`、`committed_transcript`、`committed_transcript_with_timestamps`，以及 `error/auth_error/quota_exceeded/commit_throttled/transcriber_error/unaccepted_terms/rate_limited/input_error/queue_overflow/resource_exhausted/session_time_limit_exceeded/chunk_size_exceeded/insufficient_audio_activity`（`reference/elevenlabs-python/src/elevenlabs/v_1_speech_to_text_realtime/types/receive_transcription.py:L23-L41`）。`partial_transcript` 明确“may change”，`committed_transcript_with_timestamps` 可带 `language_code` 和 `words[]`；word 字段为 `text/start/end/type/speaker_id/logprob/characters`（`reference/elevenlabs-python/src/elevenlabs/types/partial_transcript_payload.py:L10-L23`，`committed_transcript_with_timestamps_payload.py:L11-L36`，`transcription_word.py:L11-L49`）。仍未找到：官方页面对 session 最大时长、chunk 大小阈值数值的完整说明；只在错误事件中确认存在相关错误类型。

### Fish Audio

来源：官方 API 文档 https://docs.fish.audio/api-reference/endpoint/openapi-v1/speech-to-text（2026-05-09，L147-L183）。形态为 BETA REST `POST https://api.fish.audio/v1/asr`，认证为 `Authorization: Bearer <token>`（L148-L158）。文档写“只接受 application/form-data 和 application/msgpack”（L147），但示例 curl 显示 `Content-Type: application/json`（L54-L63,L110-L119），需要进一步确认真实接受类型。

请求体字段：`audio` file required，`language` string|null，`ignore_timestamps` boolean default true（L163-L183）。响应 JSON：`text` string、`duration` number seconds、`segments[]`，segment 含 `text/start/end`（L184-L210）。未找到：流式 ASR、diarization、置信度、错误对象细节、语言完整列表。

### Azure Speech to Text

来源：Microsoft Learn REST 总览 https://learn.microsoft.com/en-us/azure/ai-services/speech-service/rest-speech-to-text（2026-05-09，L32-L54）和 fast transcription guide https://learn.microsoft.com/nb-no/azure/ai-services/speech-service/fast-transcription-create（2026-05-09，L76-L133）。Azure 产品线包含 fast transcription、batch transcription、custom speech；REST API 最新 GA 版本为 `2025-10-15`（REST 总览 L36-L47）。认证可用 `Ocp-Apim-Subscription-Key` 或推荐 Entra ID Bearer token（fast guide L120-L126），endpoint 带 region。

输入：fast transcription 单文件请求，可 inline upload 或 public URL（fast guide L80-L93）；文件限制小于 5 小时、500 MB，支持 WAV/MP3/OPUS/OGG/FLAC/WMA/AAC/ALAW/MULAW/AMR/WebM/SPEEX（L76-L78）。SDK realtime 默认 WAV 16k/8k 16-bit mono PCM，部分语言 SDK 通过 GStreamer 支持 MP3/OPUS/FLAC/ALAW/MULAW/MP4 container（https://learn.microsoft.com/en-us/azure/ai-services/speech-service/how-to-use-codec-compressed-audio-input-streams，L40-L48,L162-L171）。

### NVIDIA NeMo / Parakeet

来源：clone `reference/NeMo` commit `b30e628a778089fff376ca98b6443f7eaa9f8a5c`，以及模型卡 URL https://huggingface.co/nvidia/parakeet-tdt-0.6b-v3（Web 复查，2026-05-09）。`TranscribeConfig` 字段含 `use_lhotse=True/batch_size=4/return_hypotheses=False/num_workers/channel_selector/augmentor/timestamps/verbose`（`reference/NeMo/nemo/collections/asr/parts/mixins/transcription.py:L57-L66`）。

通用 `transcribe(audio: str|list[str]|np.ndarray|DataLoader, use_lhotse=True, batch_size=4, return_hypotheses=False, num_workers=0, channel_selector=None, augmentor=None, verbose=True, timestamps=None, override_config=None, **config_kwargs)`；可传路径、列表、numpy、DataLoader；建议每文件 5-25 秒，但内存允许时可数小时；`channel_selector` 可选单通道、多通道列表或 `'average'`；timestamps 进入 Hypothesis `output.timestep['segment'/'word']`（`transcription.py:L199-L255`）。Hybrid RNNT/CTC 对 timestamps 要求 decoder 为 `ctc` 或 `rnnt`，请求 timestamps 会强制 `return_hypotheses=True` 并启用 `compute_timestamps`（`hybrid_rnnt_ctc_models.py:L95-L178`）。

Parakeet v3 模型卡已确认：600M FastConformer-TDT，自动检测并转写 25 个欧洲语言；支持语言为 Bulgarian(bg)、Croatian(hr)、Czech(cs)、Danish(da)、Dutch(nl)、English(en)、Estonian(et)、Finnish(fi)、French(fr)、German(de)、Greek(el)、Hungarian(hu)、Italian(it)、Latvian(lv)、Lithuanian(lt)、Maltese(mt)、Polish(pl)、Portuguese(pt)、Romanian(ro)、Slovak(sk)、Slovenian(sl)、Spanish(es)、Swedish(sv)、Russian(ru)、Ukrainian(uk)（HF model card L93-L99）。输入是 16kHz mono `.wav`/`.flac`，输出是带标点和大小写的 text（L156-L162）。特性含 word/segment timestamps，长音频在 A100 80GB full attention 下可到 24 分钟，local attention 可到 3 小时（L100-L107）；timestamps 调用示例返回 char/word/segment 三层（L187-L197）。模型卡给出 Parakeet streaming 方式：使用 NeMo `speech_to_text_streaming_infer_rnnt.py`，参数包括 `right_context_secs/chunk_secs/left_context_secs/batch_size`（L207-L220）。仍未找到：该脚本返回事件 schema 的稳定 API 承诺，以及 NeMo 是否把 Parakeet streaming 暴露为高层 SDK 对象。

## 4. 不可调和的设计分歧

1. 输入形态不可统一为单一类型：OpenAI/ElevenLabs/Fish 接受 multipart file，Google/AWS/Azure batch 强绑定云存储 URL，FunASR/Qwen/Whisper 接受 numpy，本地库通常不接受 HTTP multipart。
2. 流式语义不同：OpenAI Audio 是 SSE 文本 delta，OpenAI Realtime 是会话事件，Google/AWS 是 gRPC/HTTP2 双向流，腾讯/火山/阿里是自定义 WebSocket，Qwen/FunASR 是库内 incremental state。
3. partial revision 语义不同：Google/AWS 明确 interim 会修订；OpenAI SSE 是 delta append/done；腾讯用 `slice_type`；Qwen streaming 会回滚 prefix token 重解。
4. 语言码不可直接共用：OpenAI/Groq 用 ISO-639-1/Whisper 码，Google/Azure 用 BCP-47，腾讯用 `16k_zh` 这类模型码，Qwen 用语言名，sherpa SenseVoice 用 `zh/en/yue` 等短码。
5. 时间单位不统一：OpenAI/ElevenLabs/Fish/Whisper/faster-whisper 多为秒浮点，Google 是 protobuf duration string，AWS batch 常为字符串秒，Azure legacy JSON 常见 tick/offset，腾讯/阿里多处用毫秒。
6. 返回 shape 随 flag 改变：OpenAI `response_format` 可变成纯文本/SRT/VTT；ElevenLabs multi-channel 返回 `transcripts[]`；NeMo `return_hypotheses` 改变返回对象；faster-whisper 返回 iterable。
7. 高级功能不是简单 boolean：热词可能是字符串、权重表、资源 ID、自定义类、prompt；diarization 可能是 batch-only、stream-only、speaker role 或 known speaker reference。
8. job 生命周期差异大：AWS/Azure/腾讯/火山/阿里常是 submit/query/webhook；OpenAI/Groq/Fish 多是同步；ElevenLabs 同 endpoint 可 webhook early return。

## 5. 事实收敛设计

1. 最小响应几乎都能映射为 `text` + optional `segments`。
2. 主流云和 Whisper 系时间戳都可抽象为相对音频起点的 start/end，精度作为 provider capability。
3. 流式都需要表达 interim/final 或 delta/done；标准应有 `partial`, `final`, `done`, `error` 事件类别。
4. 多数 provider 支持语言可省略以自动检测，但返回语言的稳定性和格式不同，应作为 optional detected_language。
5. `words[]` 是常见扩展，但置信度、speaker、channel、char timestamp 都必须 optional。
6. 认证、region、resource id 不应进入 transcribe 核心请求；应留在 provider config。
7. `prompt/context/hotwords/custom vocabulary` 应分层：自由文本 prompt、短语 hints、预注册 vocabulary/model id。
8. 标点、ITN、脏词过滤、数字转换是常见后处理开关，但默认值差异大，需要三态：unset/provider default/on/off。
